1. 是什么？

   * 是一个类似 trae 的一个终端对话框，你可以在这个框内通过与ai对话，来完成你的项目。非翻墙完全可以由 trae 来替代

     

2. 能做什么？

   * 从0-1开发项目，通过对话方式，做出一个产品
   * 能通过对话的方式，完成git 的相关操作，存档、回滚等；

   

3. 怎么用？

   * 通过consor（这个是vscode吗） 安装 Claude code 插件；

   

4. 有哪些技巧？

   * 狂飙模式（YOLO模式），给到完全权限。

     ```shell 
     # 终端环境下
     claude --dangerously-skip-permissions
     ```

     



# 一个实践的demo

1. 目标

   基于Claude code / trae来实现一个项目。叫做model_monitor ，包含两大块，模型区分度、稳定性。

   * 入参 df[pd.DataFrame ]，包括模型的分数，入模特征，时间戳，标签dpd3【充分表现有值，否则为空】

   * 项目可以给到我，总体指标。每周模型分数的区分度 ks，稳定性psi 分箱标准以上线第一周为base等频十等分箱

   * 细分指标为模型排序性。

   * 特征则表现出稳定性即可。

     要求有每周总体psi，分箱标准也仍是第一周数据为准，同时保存分箱统计明细。





2. 过程记录

   * 生成一个数据集

     > 帮我生成一个模型分数数据，要求有模型分数score（取值300～1000的随机整数），特征值，3个即可，暂时以fea1, fea2, fea3命名，值随便，数值型即可，还有一个10位时间戳列，命名为verify_time 然后给他们打上标签（0/1），数据.csv文件，生成一个data文件夹，放进去即可。  要求数据包含近四周，并且每周数据量不低于3000

   * 基于生成的模拟数据集，开发模型监控部份
